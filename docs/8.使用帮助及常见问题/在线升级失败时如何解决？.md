## 说明
在线升级功能依赖GitHub API及相关服务，如网络不可达时则会出现检查更新失败、在线升级拉取git失败、安装新依赖包失败（如有新依赖包需要安装时）等情况出现。

如出现以上在线升级失败的情况，可通过以下方式进行更新：

1、使用Docker Copilot 等类似工具直接对Docker容器镜像进行更新。**（推荐）**

2、手动更换Github代理服务器，并执行命令进行更新。**（推荐）**

3、手动重新拉取最新Docker镜像进行更新，配置时使用原容器的配置文件路径映射到新容器/config目录，则可以保留以前的配置数据。

## 手动更换Github代理服务器，并执行命令进行更新

1、获取Github 加速代理地址，浏览器访问：https://github.akams.cn/ 代理加速站点：

![升级1](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=e7a92dc80e7833d6b6cba12b8f97a96b "升级1")

2、然后填写本项目的仓库地址：`https://github.com/smysong/mediamaster-v2.git`

![升级2](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=ad62fffdeb14e46b56c17b0b4af77692 "升级2")

3、复制加速后的仓库地址：

![升级3](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=23ffeca8da92c4c2a262d78065246ae0 "升级3")

4、打开Docker容器SSH命令窗口，执行`git remote set-url origin 加速后的地址`

![设置代理](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=c6b0495e3dde24b73b44e8d707802b5b "设置代理")

5、验证是否设置成功：

执行命令验证是否修改成功：`git remote -v`

应该看到类似输出：
```shell

origin  https://github.dpik.top/https://github.com/smysong/mediamaster-v2.git (push)
```

6、执行命令进行更新操作（三个步骤都要执行）

（1）执行命令`git pull`拉取更新文件

![Git Pull](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=bc01f2fa2eb4d510f19d56709726e2f7 "Git Pull")

（2）执行命令检查Python依赖是否有新依赖需要安装或更新，执行以下命令即可：

`pip install -r requirements.txt --index-url https://mirrors.aliyun.com/pypi/simple/`

![依赖检查1](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=92e894e3dab8821071fbacc86d5af266 "依赖检查1")

**（3）以上步骤执行完成后重启Docker容器**

**重启容器后，登录系统检查版本号是否发生变化，发生变化则成功完成更新。**

## 特殊情况
1、出现以下报错：
`error: Your local changes to the following files would be overwritten by merge:`
该提示是由于Docker容器中/app目录中的文件发生了变化或是修改了/app目录中的文件导致在线升级失败，可SSH登录容器后执行：`git checkout .`命令放弃本地更改；然后重新执行在线升级或手动拉取更新。

2、出现以下报错：
```shell
error: The following untracked working tree files would be overwritten by merge:
        movie_tvshow_btys.py
        templates/tv_alias_edit.html
        templates/tv_alias_list.html
Please move or remove them before you merge.
Aborting
```
该提示是由于本地/app目录中有一些**未被 Git 跟踪（untracked）**的文件，这些文件会和远程更新的内容冲突，Git 不敢贸然覆盖它们，因此中止了操作。
可SSH登录到容器后执行：
```shell
删除对应文件（以下文件仅为示例，请根据实际情况删除对应文件）
rm movie_tvshow_btys.py
rm templates/tv_alias_edit.html
rm templates/tv_alias_list.html
```
然后重新执行在线升级或手动拉取更新。
