
v2.5.8版本新增支持通过环境变量配置WEB服务器的运行端口。

自定义端口号功能主要针对于**Host模式**部署时可以修改WEB服务的端口号，具体配置方法如下：

#### Docker Compose配置

修改Docker Compose配置，添加环境变量：`- PORT=9999` 该变量指定WEB服务的运行端口号为9999

![自定义端口号3](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=1613300832227b9034ed46a10e7ab632 "自定义端口号1")

Host模式使用自定义端口号进行访问

![自定义端口号4](http://wiki.songmy.top:8080/server/index.php?s=/api/attachment/visitFile&sign=13410634cc19d2391870d729448f3deb)

#### 命令行方式配置

```
docker run -d \
  --name mediamaster \
  --network=host \
  -v /path/to/config:/config \
  -v /Media:/Media \
  -v /Torrent:/Torrent \
  -v /Downloads:/Downloads \
  -e PORT=9999 \ 
  docker.1ms.run/smysong/mediamaster-v2:latest
```

### GUI方式配置

在Docker管理器中修改容器配置，添加自定义端口环境变量
